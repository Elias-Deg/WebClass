# Mentorship Kit Overview

This folder contains supporting Markdown files for the mentorship plan.

For the easiest teaching experience, use the local dashboard instead:

- [Mentor Portal](../mentor-portal/index.html)

The portal runs as a plain HTML, CSS, and JavaScript website and includes the day-by-day lessons, videos, homework, tests, progress tracking, and mentor notes.

The full plan assumes 5 learning days per week, with each day containing:

- Mentor discussion
- Simple analogy
- Video or reading assignment
- Research prompt
- Practical exercise
- Homework
- Checkpoint question

## Recommended Rhythm

Each mentorship day should follow this pattern:

1. **Warm-up:** Ask what they remember from the previous day.
2. **Concept:** Explain today's topic with a simple analogy.
3. **Watch:** Assign the relevant video section or resource.
4. **Demo:** Build a tiny example while explaining your thinking.
5. **Practice:** Let them build with support.
6. **Homework:** Give them a small independent task.
7. **Review:** End with 2-3 questions to confirm understanding.

## Folder Guide

- `../mentor-portal/`: User-friendly dashboard for teaching the full curriculum.
- `video-recommendations.md`: YouTube videos and official documentation links.
- `homework-tracker.md`: Progress tracker you can duplicate per mentee.
- `mentor-session-template.md`: Repeatable structure for each mentorship session.

## How To Grade Progress

Use three levels:

- **Needs support:** They can only complete the task while being guided step by step.
- **Developing:** They can complete the task with hints, documentation, or debugging help.
- **Ready for next phase:** They can explain the idea and build a small working example independently.

## Core Rule

Do not move forward just because the day is finished. Move forward when they can explain the idea in plain language and use it in a tiny project.

## Final Target

The mentee is ready for supervised junior work when they can complete this task:

> Add a protected dashboard page to a Next.js App Router app where logged-in users can create, view, update, and delete their own tasks, then open a pull request and deploy it to a preview environment.
